\begin{abstract}
Estas notas explican matematicamente el codigo del repositorio
\code{nicolas-lm}. El documento no es una introduccion generica a deep learning:
cada definicion se conecta con un modulo concreto del codigo, con las
dimensiones de sus tensores, con los parametros que aprende y con el experimento
de diversidad textual. La exposicion usa como guias a Petersen--Zech para
probabilidad, riesgo empirico, optimizacion, redes y Transformers
\cite{petersen-zech-2026-mathematical}; a Wegner para datasets, regresion,
maxima verosimilitud, alta dimension, SVD, perceptron, redes y descenso de
gradiente \cite{wegner-2024-mathematical-introduction-data-science}; a Calin
para activaciones, costos, entropia cruzada, divergencias, backpropagation,
informacion, capacidad y arquitectura \cite{calin-2020-deep-learning-architectures};
y a Chang et al. para la interpretacion de calidad de datos y tokens efectivos
\cite{chang-etal-2024-scaling}.
\end{abstract}

\section{Fundamentos de probabilidad}

Petersen--Zech colocan el aprendizaje supervisado sobre una distribucion de
datos \(\calD\). Para que esa frase sea matematica, primero se requiere el
lenguaje de espacios medibles y variables aleatorias.

\subsection{Espacios de probabilidad}

\begin{definition}[Sigma-algebra]
Sea \(\Omega\) un conjunto. Una familia \(\calA\subseteq 2^\Omega\) es una
sigma-algebra si:
\[
\Omega\in\calA,\qquad A\in\calA\Rightarrow A^c\in\calA,\qquad
A_1,A_2,\ldots\in\calA\Rightarrow \bigcup_{i=1}^\infty A_i\in\calA.
\]
El par \((\Omega,\calA)\) se llama espacio medible.
\end{definition}

\begin{definition}[Probabilidad]
Una probabilidad sobre \((\Omega,\calA)\) es una medida
\(\Prob:\calA\to[0,1]\) tal que \(\Prob(\Omega)=1\) y, para eventos disjuntos
\((A_i)_{i\ge 1}\),
\[
\Prob\left(\bigcup_{i=1}^{\infty}A_i\right)=
\sum_{i=1}^{\infty}\Prob(A_i).
\]
\end{definition}

\begin{remark}[Interpretacion para lenguaje]
En un modelo idealizado, \(\omega\in\Omega\) puede representar una realizacion
completa de una fuente textual. El corpus observado en el experimento no es la
distribucion misma, sino una secuencia finita producida por algun mecanismo
desconocido.
\end{remark}

\subsection{Medidas, espacios finitos y ley de una variable}

Los apuntes locales \code{capitulo04.tex} desarrollan primero teoria de la
medida antes de hablar de probabilidad. Esta capa es util aqui porque el codigo
manipula objetos discretos, pero el lenguaje matematico de aprendizaje usa
distribuciones, esperanzas e imagenes directas de medidas.

\begin{definition}[Medida positiva]
Sea \((E,\mathcal E)\) un espacio medible. Una medida positiva es una funcion
\[
\mu:\mathcal E\to[0,\infty]
\]
tal que \(\mu(\varnothing)=0\) y, para conjuntos medibles disjuntos
\((A_n)_{n\geq1}\),
\[
\mu\left(\bigcup_{n=1}^{\infty}A_n\right)
=
\sum_{n=1}^{\infty}\mu(A_n).
\]
El triplete \((E,\mathcal E,\mu)\) se llama espacio de medida.
\end{definition}

\begin{remark}[El caso del repositorio es discreto]
Para el vocabulario finito \(\calV=\{0,\ldots,m-1\}\), la sigma-algebra natural
es \(\mathcal P(\calV)\). Para ventanas de longitud \(B\), el espacio de estados
es \(\calV^B\) y la sigma-algebra natural es \(\mathcal P(\calV^B)\). En este
caso toda funcion \(f:\calV^B\to\R^k\) es medible. Esta observacion evita
tecnicismos, pero no elimina la necesidad conceptual de hablar de medidas:
entrenar un modelo de lenguaje es estimar distribuciones sobre estos espacios
finitos enormes.
\end{remark}

\begin{definition}[Imagen directa o pushforward]
Si \(\varphi:(E,\mathcal E)\to(F,\mathcal F)\) es medible y \(\mu\) es una
medida sobre \(E\), la imagen directa \(\varphi_\#\mu\) se define por
\[
(\varphi_\#\mu)(A)=\mu(\varphi^{-1}(A)),
\qquad A\in\mathcal F.
\]
\end{definition}

\begin{example}[La ley de una ventana tokenizada]
Si \(W:\Omega\to\calV^B\) envia una realizacion textual \(\omega\) a la ventana
\[
W(\omega)=(X_t(\omega),\ldots,X_{t+B-1}(\omega)),
\]
entonces la distribucion de la ventana es el pushforward
\[
P_W=W_\#\Prob.
\]
El dataset empirico construido por \code{TokenDataset} reemplaza esa ley
desconocida por la medida empirica
\[
\widehat P_N=\frac{1}{N-B}\sum_{i=0}^{N-B-1}\delta_{(x_i,\ldots,x_{i+B-1})}.
\]
\end{example}

\begin{definition}[Medida producto]
Si \((E,\mathcal E,\mu)\) y \((F,\mathcal F,\nu)\) son espacios de medida
sigma-finitos, la medida producto \(\mu\otimes\nu\) sobre
\((E\times F,\mathcal E\otimes\mathcal F)\) queda caracterizada por
\[
(\mu\otimes\nu)(A\times B)=\mu(A)\nu(B).
\]
\end{definition}

\begin{remark}[Por que importa el producto]
La formulacion iid clasica \(S\sim\calD^n\) usa exactamente una medida producto:
\[
\calD^n=\underbrace{\calD\otimes\cdots\otimes\calD}_{n\text{ veces}}.
\]
En el repo, las ventanas provienen de una misma trayectoria textual y no de una
extraccion producto independiente. Esta diferencia es la razon formal por la
cual las cotas iid de generalizacion deben leerse como marco conceptual, no como
certificado exacto para las metricas reportadas.
\end{remark}

\subsection{Variables aleatorias y distribuciones}

\begin{definition}[Variable aleatoria]
Si \((\Omega,\calA)\) y \((E,\mathcal E)\) son espacios medibles, una funcion
\(X:\Omega\to E\) es una variable aleatoria si
\[
X^{-1}(B)\in\calA
\qquad\text{para todo }B\in\mathcal E.
\]
\end{definition}

\begin{definition}[Distribucion]
La distribucion de \(X\) es la medida \(P_X\) sobre \(E\) dada por
\[
P_X(B)=\Prob(X\in B).
\]
Equivalente y mas estructuralmente,
\[
P_X=X_\#\Prob.
\]
\end{definition}

\begin{example}[Tokens]
Si el vocabulario del tokenizador tiene tamano \(m\), cada token puede
representarse como una variable aleatoria con valores en
\[
\calV=\{0,\ldots,m-1\}.
\]
Una secuencia textual se modela entonces como un proceso discreto
\[
X_0,X_1,\ldots,X_{N-1},\qquad X_t:\Omega\to\calV.
\]
\end{example}

\subsection{Esperanza, varianza e independencia}

\begin{definition}
Si \(X:\Omega\to\R^d\) es integrable, su esperanza es
\[
\E[X]=\int_\Omega X(\omega)\,d\Prob(\omega).
\]
Si \(X:\Omega\to\R\), su varianza es
\[
\Var(X)=\E[(X-\E[X])^2].
\]
Si \(X,Y\) son reales, su covarianza es
\[
\Cov(X,Y)=\E[(X-\E[X])(Y-\E[Y])].
\]
\end{definition}

\begin{remark}[Esperanza como integral de Lebesgue]
\code{capitulo04.tex} construye la integral empezando por funciones simples y
luego pasando a funciones medibles no negativas por aproximacion monotona. En
este proyecto casi todo es finito: para una variable \(X\) con valores
\(\{a_1,\ldots,a_r\}\),
\[
\E[X]=\sum_{j=1}^r a_j\Prob(X=a_j).
\]
La notacion integral es, sin embargo, la correcta para conectar con riesgo
poblacional, leyes de variables aleatorias y resultados de aprendizaje
estadistico.
\end{remark}

\begin{proposition}[Linealidad y Cauchy--Schwarz]
Si \(X,Y\) son integrables y \(a,b\in\R\), entonces
\[
\E[aX+bY]=a\E[X]+b\E[Y].
\]
Si \(X,Y\in L^2(\Prob)\), entonces
\[
|\E[XY]|\leq \left(\E[X^2]\right)^{1/2}
\left(\E[Y^2]\right)^{1/2}.
\]
\end{proposition}

\begin{remark}
La segunda desigualdad es el caso \(p=q=2\) de Holder, presentado en
\code{capitulo04.tex}. En optimizacion aparece como Cauchy--Schwarz para
justificar que \(-\nabla F\) es direccion de descenso local; en estadistica
controla productos e interacciones entre variables aleatorias.
\end{remark}

\begin{definition}[Independencia]
Dos variables \(X,Y\) son independientes si para todo par de conjuntos
medibles \(A,B\),
\[
\Prob(X\in A,Y\in B)=\Prob(X\in A)\Prob(Y\in B).
\]
\end{definition}

\begin{remark}[Ventanas no iid]
La independencia es precisamente lo que falla en \code{TokenDataset}. Si
\[
X_i=(x_i,\ldots,x_{i+B-1}),\qquad
X_{i+1}=(x_{i+1},\ldots,x_{i+B}),
\]
entonces \(X_i\) y \(X_{i+1}\) comparten \(B-1\) tokens. Por eso el dataset de
lenguaje no debe tratarse como una muestra iid ordinaria. Esta advertencia es
importante cuando \code{scripts/evaluate.py} reporta desviacion estandar, error
estandar e intervalo descriptivo: esos numeros resumen variabilidad de batches,
pero no son intervalos iid formales.
\end{remark}

\subsection{Distribuciones condicionales y modelado autoregresivo}

\begin{definition}[Probabilidad condicional discreta]
Si \(\Prob(Y=y)>0\), entonces
\[
\Prob(X=x\mid Y=y)
=
\frac{\Prob(X=x,Y=y)}{\Prob(Y=y)}.
\]
\end{definition}

\begin{definition}[Modelo autoregresivo]
Un modelo de lenguaje autoregresivo factoriza la distribucion de una secuencia
\((x_0,\ldots,x_{T-1})\) como
\[
p_\theta(x_0,\ldots,x_{T-1})
=
\prod_{t=0}^{T-1}p_\theta(x_t\mid x_0,\ldots,x_{t-1}).
\]
En entrenamiento de next-token prediction se estima, para cada posicion,
\[
p_\theta(x_{t+1}\mid x_{\le t}).
\]
\end{definition}

\subsection{Entropia, entropia condicional y maxima verosimilitud}

\begin{definition}[Entropia discreta]
Si \(X\) toma valores en un conjunto finito y \(p_i=\Prob(X=i)\), su entropia
es
\[
H(X)=-\sum_i p_i\log p_i.
\]
\end{definition}

\begin{definition}[Entropia condicional]
Para variables discretas,
\[
H(Y\mid X)
=
-\sum_x\sum_y \Prob(X=x,Y=y)\log \Prob(Y=y\mid X=x).
\]
\end{definition}

\begin{definition}[Verosimilitud]
Dadas observaciones \(z_1,\ldots,z_n\) y un modelo parametrico
\(p_\theta(z)\), la verosimilitud es
\[
L(\theta)=\prod_{i=1}^n p_\theta(z_i).
\]
Maximizar \(L\) equivale a minimizar la negative log-likelihood
\[
-\log L(\theta)
=
\sum_{i=1}^n -\log p_\theta(z_i).
\]
\end{definition}

\begin{remark}
Wegner usa la maxima verosimilitud para conectar regresion y supuestos
probabilisticos. En modelos de lenguaje, la entropia cruzada implementada por
\code{F.cross\_entropy} es exactamente una negative log-likelihood categorica:
se penaliza al modelo por asignar poca probabilidad al token observado.
\end{remark}

\subsection{Convergencia e intercambio de limites}

La teoria de la medida de \code{capitulo04.tex} tambien explica por que los
valores esperados se comportan bien bajo limites. En aprendizaje esto aparece
cuando se reemplaza una distribucion por medidas empiricas, cuando se promedian
perdidas y cuando se justifica pasar de sumas finitas a riesgos esperados.

\begin{proposition}[Convergencia dominada, forma util]
Sea \(Z_n\to Z\) casi seguramente y supongase que existe \(G\in L^1(\Prob)\)
tal que \(|Z_n|\leq G\) casi seguramente para todo \(n\). Entonces
\[
\lim_{n\to\infty}\E[Z_n]=\E[Z].
\]
\end{proposition}

\begin{remark}[Uso conceptual en estas notas]
No necesitamos probar convergencia dominada para ejecutar el repo. Pero si
queremos explicar rigurosamente por que las perdidas promedio son aproximaciones
de riesgos esperados, la teoria de integracion es el lenguaje adecuado. En el
caso finito del tokenizer, muchas integrales se reducen a sumas; aun asi, el
marco medible evita confundir una suma empirica con la distribucion generadora.
\end{remark}

\section{Estadistica y aprendizaje estadistico}

\subsection{Datasets, hipotesis y distribucion de datos}

\begin{definition}[Dataset etiquetado]
Sean \(X\) el espacio de entradas y \(Y\) el espacio de salidas. Un dataset
etiquetado es una coleccion finita
\[
S=\{(x_i,y_i)\}_{i=1}^n\subseteq X\times Y.
\]
\end{definition}

\begin{definition}[Hipotesis]
Una clase de hipotesis es un conjunto \(\calH\) de funciones \(h:X\to Y\).
En deep learning, \(\calH\) suele estar parametrizada:
\[
\calH=\{h_\theta:\theta\in\R^p\}.
\]
\end{definition}

\begin{definition}[Riesgo poblacional]
Sea \(\calD\) una distribucion sobre \(X\times Y\) y sea
\(\ell:Y\times Y\to\R_+\) una perdida. El riesgo de \(h\) es
\[
R(h)=\E_{(x,y)\sim\calD}[\ell(h(x),y)].
\]
\end{definition}

\begin{definition}[Riesgo empirico]
Para un dataset \(S=\{(x_i,y_i)\}_{i=1}^n\),
\[
\widehat R_S(h)=\frac1n\sum_{i=1}^n \ell(h(x_i),y_i).
\]
Un minimizador de riesgo empirico satisface
\[
h_S\in\argmin_{h\in\calH}\widehat R_S(h).
\]
\end{definition}

\subsection{El dataset de \texttt{nicolas-lm}}

El espacio de entrada del repositorio no es \(\R^d\) inicialmente. Primero hay
texto, luego caracteres, luego ids enteros, y finalmente tensores.

\begin{definition}[Secuencia tokenizada]
Sea \(c_0c_1\cdots c_{N-1}\) un texto. El tokenizador produce
\[
x_t=\tau(c_t)\in\{0,\ldots,m-1\},
\]
donde \(m\) es el tamano del vocabulario.
\end{definition}

\begin{definition}[Ventanas de entrenamiento]
Para block size \(B\), \code{TokenDataset} define
\[
X_i=(x_i,x_{i+1},\ldots,x_{i+B-1})\in\calV^B,
\]
\[
Y_i=(x_{i+1},x_{i+2},\ldots,x_{i+B})\in\calV^B.
\]
El numero total de ejemplos de una secuencia de longitud \(N\) es \(N-B\).
\end{definition}

\begin{remark}[Dimensiones en codigo]
El metodo \code{get\_batch(batch\_size)} devuelve listas que luego
\code{batch\_to\_tensors} transforma en tensores
\[
x,y\in\{0,\ldots,m-1\}^{M\times B},
\]
donde \(M\) es el batch size. En el experimento principal,
\[
B=64,\qquad M=32.
\]
\end{remark}

\subsection{Train, validation y test}

\begin{definition}
\code{train\_val\_test\_split} divide una secuencia en tres bloques contiguos:
\[
\text{train}=x_{0:n_1-1},\qquad
\text{val}=x_{n_1:n_2-1},\qquad
\text{test}=x_{n_2:N-1},
\]
donde
\[
n_1=\lfloor \alpha N\rfloor,\qquad
n_2=\lfloor(\alpha+\beta)N\rfloor.
\]
Por defecto, \(\alpha=0.8\) y \(\beta=0.1\).
\end{definition}

\begin{remark}
Train actualiza parametros. Validation monitorea entrenamiento y
hiperparametros. Test queda reservado para evaluacion final. El hecho de que
los bloques sean contiguos hace simple el protocolo, pero puede introducir
desplazamientos de distribucion si el texto cambia de estilo a lo largo del
corpus.
\end{remark}

\subsection{Generalizacion, capacidad y overfitting}

\begin{definition}[Brecha de generalizacion]
En el reporte del repo,
\[
\mathrm{gap}
=
\mathrm{test\ loss}-\mathrm{train\ loss}.
\]
Una brecha positiva indica que el modelo ajusta mejor el conjunto usado para
entrenar que el bloque reservado para test.
\end{definition}

\begin{remark}[Capacidad]
Petersen--Zech formulan la tension entre error de aproximacion y error de
generalizacion. En este proyecto, el bigrama tiene baja capacidad; el
Transformer y el LLaMA-style tienen mayor capacidad. Mayor capacidad permite
representar dependencias mas complejas, pero tambien puede exigir mas datos y
mas optimizacion.
\end{remark}

\subsection{Riesgo de Bayes, ERM y descomposicion de generalizacion}

\code{capitulo03 (1).tex} organiza la teoria de generalizacion alrededor de una
pregunta precisa: si \(h_S\) minimiza el riesgo empirico, cuanto podemos decir
del riesgo poblacional \(R(h_S)\)? Primero se define el punto de referencia
ideal.

\begin{definition}[Riesgo de Bayes]
El riesgo de Bayes es
\[
R^\star=\inf_{h:X\to Y}R(h),
\]
donde el infimo se toma sobre todas las funciones medibles admisibles. Si se
restringe a una clase \(\calH\), aparece el mejor riesgo alcanzable dentro de
esa clase:
\[
R^\star_{\calH}=\inf_{h\in\calH}R(h).
\]
\end{definition}

\begin{proposition}[Descomposicion ERM]
Sea \(h_S\in\calH\) un minimizador del riesgo empirico y sea
\[
\varepsilon_{\mathrm{gen}}
=
\sup_{h\in\calH}|R(h)-\widehat R_S(h)|.
\]
Entonces
\[
R(h_S)-R^\star
\leq
2\varepsilon_{\mathrm{gen}}
+
\left(\inf_{g\in\calH}R(g)-R^\star\right).
\]
\end{proposition}

\begin{proof}
Para cualquier \(g\in\calH\),
\[
R(h_S)-R^\star
=
R(h_S)-\widehat R_S(h_S)
+\widehat R_S(h_S)-R^\star.
\]
Como \(h_S\) minimiza \(\widehat R_S\),
\(\widehat R_S(h_S)\leq \widehat R_S(g)\). Por tanto,
\[
R(h_S)-R^\star
\leq
|R(h_S)-\widehat R_S(h_S)|
+\widehat R_S(g)-R^\star.
\]
Sumando y restando \(R(g)\),
\[
R(h_S)-R^\star
\leq
2\varepsilon_{\mathrm{gen}}+R(g)-R^\star.
\]
Tomar infimo sobre \(g\in\calH\) da la afirmacion.
\end{proof}

\begin{remark}[Lectura para este repo]
El primer termino mide generalizacion; el segundo mide aproximacion. En
\code{nicolas-lm}, el bigrama puede tener gran error de aproximacion porque su
historia efectiva es solo \(x_t\). El Transformer y LLaMA-style reducen ese sesgo
al usar contexto, pero pueden aumentar dificultad de estimacion y optimizacion.
Esta es la forma matematica de decir que ``mas arquitectura'' no garantiza
automaticamente menor test loss.
\end{remark}

\subsection{Cotas de generalizacion y clases finitas}

\begin{definition}[Cota de generalizacion]
Una funcion \(\kappa(\delta,n)\) es una cota de generalizacion para \(\calH\)
si, con probabilidad al menos \(1-\delta\) sobre \(S\sim\calD^n\),
\[
\sup_{h\in\calH}|R(h)-\widehat R_S(h)|
\leq
\kappa(\delta,n).
\]
\end{definition}

\begin{proposition}[Hoeffding mas union bound]
Supongase que \(\calH=\{h_1,\ldots,h_N\}\) es finita y que
\(\ell(h(x),y)\in[c_1,c_2]\), con \(C=c_2-c_1\). Si
\(S\sim\calD^n\) es iid, entonces con probabilidad al menos \(1-\delta\),
\[
\sup_{h\in\calH}|R(h)-\widehat R_S(h)|
\leq
C\sqrt{\frac{\log|\calH|+\log(2/\delta)}{2n}}.
\]
\end{proposition}

\begin{remark}[Por que no se aplica literalmente]
Esta cota usa dos supuestos fuertes: clase finita y muestra iid. Nuestros
modelos tienen parametros continuos, y las ventanas del dataset estan
solapadas. Aun asi, la formula muestra la intuicion correcta: la generalizacion
empeora cuando crece la complejidad efectiva de \(\calH\), y mejora cuando crece
el numero de observaciones efectivamente independientes.
\end{remark}

\subsection{Numeros de cubrimiento}

\begin{definition}[Numero de cubrimiento]
Sea \(A\) un subconjunto de un espacio metrico \((M,d)\). El numero de
cubrimiento por radio \(\varepsilon\) es
\[
\mathcal G(A,\varepsilon,(M,d))
=
\min\left\{
N:\exists z_1,\ldots,z_N\in M
\text{ con }
A\subseteq \bigcup_{i=1}^N B_\varepsilon(z_i)
\right\}.
\]
\end{definition}

\begin{remark}[Interpretacion]
Un cubrimiento reemplaza una clase infinita por una grilla finita de
representantes. La teoria de \code{capitulo03 (1).tex} usa esta idea para pasar
de la cota de Hoeffding para clases finitas a clases de redes neuronales con
parametros continuos. Si la aplicacion de realizacion
\[
\theta\mapsto h_\theta
\]
es Lipschitz en un dominio acotado de parametros, entonces cubrir el espacio de
parametros cubre tambien la familia de funciones.
\end{remark}

\begin{proposition}[Cubrimiento de una caja de parametros]
Para \(B,\varepsilon>0\),
\[
\mathcal G([-B,B]^p,\varepsilon,\|\cdot\|_\infty)
\leq
\left\lceil\frac{B}{\varepsilon}\right\rceil^p.
\]
\end{proposition}

\begin{remark}[Conexion con parametros del repo]
El numero \(p\) no es solo una cifra administrativa: corresponde al numero de
grados de libertad aprendibles. Para el bigrama, \(p=m^2\). Para el Transformer
y LLaMA-style, \(p\) incluye embeddings, proyecciones de atencion, parametros de
normalizacion, MLP/SwiGLU y cabeza de lenguaje. La teoria de cubrimientos sugiere
que aumentar \(p\) aumenta la riqueza de la clase, pero tambien aumenta el
termino de complejidad que debe compensarse con mas datos efectivos.
\end{remark}

\subsection{Compromiso aproximacion-complejidad}

La lectura central de \code{capitulo03 (1).tex} para nuestro experimento es el
compromiso entre capacidad y generalizacion. Una forma esquematica de la cota es
\[
R(\Phi_S)-R^\star
\lesssim
\underbrace{
\sqrt{\frac{sL\log(mBs)}{n}}
}_{\text{complejidad/generalizacion}}
+
\underbrace{s^{-\alpha}}_{\text{aproximacion}},
\]
donde \(s\) representa tamano efectivo de red, \(L\) profundidad, \(B\) cota de
pesos, \(n\) tamano muestral y \(\alpha\) una tasa de aproximacion idealizada.

\begin{remark}[Traduccion al experimento high vs medium]
Si el corpus high aumenta diversidad local, tambien puede aumentar la
complejidad de la funcion condicional
\[
x_{\leq t}\mapsto p(x_{t+1}\mid x_{\leq t}).
\]
Entonces el termino de aproximacion puede mejorar al pasar de bigrama a
Transformer, pero el termino de generalizacion/optimizacion puede empeorar si el
numero de tokens efectivos no crece lo suficiente. Por eso ``high es mas
diverso'' no implica automaticamente ``high produce menor test loss''.
\end{remark}

\begin{definition}[Subajuste y sobreajuste en estas notas]
Diremos que hay subajuste cuando la perdida de train y test son ambas altas:
el modelo no expresa o no optimiza bien la estructura del corpus. Diremos que
hay sobreajuste cuando train loss es bajo pero test loss aumenta
sustancialmente, es decir, cuando \(\mathrm{gap}\) domina la interpretacion.
\end{definition}

\subsection{Dimension VC y advertencia sobre sobreparametrizacion}

\begin{definition}[Fragmentacion y dimension VC]
Para una clase binaria \(\calH\subseteq\{h:X\to\{0,1\}\}\), un conjunto
\(\{x_1,\ldots,x_k\}\) es fragmentado por \(\calH\) si para todo vector de
etiquetas \((y_1,\ldots,y_k)\in\{0,1\}^k\) existe \(h\in\calH\) tal que
\[
h(x_i)=y_i,\qquad i=1,\ldots,k.
\]
La dimension VC es el mayor \(k\) para el cual existe un conjunto fragmentado.
\end{definition}

\begin{proposition}[Cota VC clasica]
Si \(\mathrm{VCdim}(\calH)=k\), entonces en clasificacion binaria con perdida
\(0\)-\(1\), bajo muestra iid de tamano \(n\geq k\), con probabilidad al menos
\(1-\delta\),
\[
|R(h)-\widehat R_S(h)|
\leq
\sqrt{\frac{2k\log(en/k)}{n}}
+
\sqrt{\frac{\log(1/\delta)}{2n}}
\]
simultaneamente para todo \(h\in\calH\).
\end{proposition}

\begin{remark}[Por que incluir VC si nuestro objetivo es multiclase]
El repo hace prediccion categorica con \(m\) clases, no clasificacion binaria.
La dimension VC no es la herramienta exacta para nuestro objetivo, pero si
expresa una idea que importa: la capacidad de una clase se mide por los patrones
que puede realizar, no solo por el numero bruto de parametros. Para redes
profundas, las cotas clasicas pueden sugerir pesimismo en regimenes
sobreparametrizados; por eso se usan como advertencia conceptual, no como
prediccion numerica del test loss de este experimento.
\end{remark}

\section{Optimizacion}

\subsection{De ERM a minimizacion}

Una vez elegida la familia \(p_\theta\), entrenar significa resolver
\[
\widehat\theta
\in
\argmin_{\theta\in\R^p}
\widehat R(\theta),
\]
donde \(\widehat R\) es la perdida promedio sobre batches de ventanas.

\begin{definition}[Funcion objetivo]
Para un batch \((X,Y)\in\calV^{M\times B}\times\calV^{M\times B}\), el modelo
produce logits
\[
Z_\theta(X)\in\R^{M\times B\times m}.
\]
La perdida de batch es
\[
\widehat R_B(\theta)
=
\frac{1}{MB}\sum_{a=1}^M\sum_{t=1}^B
-\log \softmax(Z_\theta(X)_{a,t,:})_{Y_{a,t}}.
\]
\end{definition}

\subsection{Gradiente y descenso de gradiente}

\begin{definition}[Gradiente]
Si \(F:\R^p\to\R\) es diferenciable,
\[
\nabla F(\theta)
=
\left(\frac{\partial F}{\partial\theta_1},\ldots,
\frac{\partial F}{\partial\theta_p}\right).
\]
Para perturbaciones pequenas,
\[
F(\theta+v)=F(\theta)+\langle\nabla F(\theta),v\rangle+O(\|v\|^2).
\]
\end{definition}

\begin{proposition}[Direccion local de descenso]
Si \(\nabla F(\theta)\neq 0\), la direccion unitaria que minimiza la
aproximacion lineal es
\[
v^\star=-\frac{\nabla F(\theta)}{\|\nabla F(\theta)\|}.
\]
\end{proposition}

\begin{proof}
Por Cauchy--Schwarz,
\[
\langle\nabla F(\theta),v\rangle\ge
-\|\nabla F(\theta)\|\|v\|.
\]
Si \(\|v\|=1\), la cota se alcanza con
\(v=-\nabla F(\theta)/\|\nabla F(\theta)\|\).
\end{proof}

\begin{definition}[Descenso de gradiente]
\[
\theta_{k+1}=\theta_k-\eta_k\nabla F(\theta_k).
\]
El escalar \(\eta_k>0\) es la tasa de aprendizaje.
\end{definition}

\subsection{SGD, minibatches y AdamW}

\begin{definition}[Gradiente estocastico]
Si
\[
F(\theta)=\frac1n\sum_{i=1}^nF_i(\theta),
\]
un minibatch \(I_k\subset\{1,\ldots,n\}\) define
\[
G_k(\theta)=\frac1{|I_k|}\sum_{i\in I_k}\nabla F_i(\theta).
\]
El update de SGD es
\[
\theta_{k+1}=\theta_k-\eta_kG_k(\theta_k).
\]
\end{definition}

\begin{definition}[Adam]
Adam mantiene promedios moviles del gradiente y del gradiente cuadrado:
\[
m_k=\beta_1m_{k-1}+(1-\beta_1)g_k,
\]
\[
v_k=\beta_2v_{k-1}+(1-\beta_2)(g_k\odot g_k).
\]
La actualizacion idealizada es
\[
\theta_{k+1}
=
\theta_k-\eta\frac{\widehat m_k}{\sqrt{\widehat v_k}+\epsilon}.
\]
\end{definition}

\begin{remark}[AdamW en \code{scripts/train.py}]
El entrenamiento usa
\[
\code{torch.optim.AdamW(model.parameters(), lr=3e-4)}.
\]
En cada paso el codigo ejecuta:
\[
\code{loss.backward()},\qquad
\code{optimizer.step()}.
\]
La primera operacion calcula gradientes por backpropagation; la segunda aplica
el update adaptativo. La `W' en AdamW indica weight decay desacoplado.
\end{remark}

\subsection{Backpropagation}

\begin{definition}[Red como composicion]
Sea
\[
f_\theta=f_L\circ f_{L-1}\circ\cdots\circ f_1.
\]
La regla de la cadena da
\[
\frac{\partial \ell(f_\theta(x),y)}{\partial\theta_\ell}
=
\frac{\partial \ell}{\partial f_L}
\frac{\partial f_L}{\partial f_{L-1}}\cdots
\frac{\partial f_{\ell+1}}{\partial f_\ell}
\frac{\partial f_\ell}{\partial\theta_\ell},
\]
con la notacion adecuada de Jacobianos.
\end{definition}

\begin{remark}
Calin y Petersen--Zech presentan backpropagation como el mecanismo que hace
computacionalmente viable entrenar redes profundas. En el repo no se implementa
a mano: PyTorch construye el grafo computacional y \code{loss.backward()} aplica
automaticamente la regla de la cadena.
\end{remark}

\section{Deep learning como familia de funciones parametrizadas}

\subsection{Neuronas, capas y activaciones}

\begin{definition}[Neurona]
Una neurona es una funcion
\[
\nu(x)=\sigma(w^\top x+b),
\]
donde \(w\in\R^d\), \(b\in\R\) y \(\sigma:\R\to\R\) es una activacion.
\end{definition}

\begin{definition}[Capa]
Una capa de ancho \(k\) es una aplicacion
\[
T:\R^d\to\R^k,\qquad
T(x)=\sigma(Wx+b),
\]
donde \(W\in\R^{k\times d}\), \(b\in\R^k\), y \(\sigma\) actua coordenada a
coordenada:
\[
T(x)_r=\sigma(\langle W_{r,:},x\rangle+b_r).
\]
Esta es la formalizacion de \code{capitulo02 (1).tex}: una capa apila \(k\)
neuronas que comparten la misma entrada.
\end{definition}

\begin{definition}[Red feedforward]
Una red feedforward es una composicion
\[
f_\theta
=
T_L\circ\sigma\circ T_{L-1}\circ\cdots\circ\sigma\circ T_1,
\]
donde \(T_\ell(z)=W_\ell z+b_\ell\).
\end{definition}

\begin{definition}[Arquitectura, parametros e hiperparametros]
Una arquitectura feedforward queda especificada por
\[
(\sigma;d_0,d_1,\ldots,d_{L+1}),
\]
donde \(d_0\) es la dimension de entrada, \(d_{L+1}\) la dimension de salida,
\(L\) el numero de capas ocultas y \(d_1,\ldots,d_L\) sus anchos. Los parametros
aprendibles son las entradas de las matrices \(W^{(\ell)}\) y sesgos
\(b^{(\ell)}\). Los hiperparametros son elecciones fijadas antes de entrenar:
profundidad, ancho, activacion, learning rate, block size, batch size y numero
de pasos.
\end{definition}

\begin{remark}[Traduccion al repo]
En \code{nicolas-lm}, \code{embedding\_dim}, \code{num\_heads},
\code{num\_layers}, \code{block\_size} y \code{dropout} son hiperparametros.
Las tablas de embedding, proyecciones de atencion, parametros de normalizacion,
matrices de SwiGLU/MLP y \code{lm\_head} son parametros. Esta distincion importa:
AdamW actualiza parametros; no aprende automaticamente el block size ni el
numero de capas.
\end{remark}

\begin{remark}[Activaciones]
Calin organiza las activaciones en familias sigmoidales, tipo ReLU y tipo bump.
En este repo aparecen \(\GELU\) en el feedforward Transformer y \(\SiLU\) dentro
de SwiGLU. Ambas introducen no linealidad suave, sin la cual una composicion de
capas lineales seguiria siendo lineal.
\end{remark}

\subsection{Tamano, peso compartido y estructura}

\code{capitulo02 (1).tex} introduce una nocion de tamano como numero de
parametros aprendibles, permitiendo peso compartido y esparsidad. Si una red
tiene pesos y sesgos reunidos en un vector \(\theta\in\R^p\), su tamano
efectivo es el numero de coordenadas libres de \(\theta\), no necesariamente el
numero de aristas visibles en un grafo.

\begin{definition}[Tamano efectivo]
En estas notas escribiremos
\[
\mathrm{size}(f_\theta)=\dim(\Theta)
\]
para el numero de grados de libertad aprendibles de la familia
\(\{f_\theta:\theta\in\Theta\}\). Si hay pesos compartidos, varias entradas de
matrices corresponden a una sola coordenada de \(\theta\). Si hay entradas
fijadas en cero, no cuentan como parametros aprendibles.
\end{definition}

\begin{remark}[Peso compartido en modelos de lenguaje]
El modelo Transformer no es una red feedforward densa generica aplicada a una
ventana aplanada. Sus matrices de atencion y feedforward se aplican a cada
posicion temporal con los mismos pesos. Formalmente, si \(W_Q\in\R^{d\times d_h}\)
define queries, entonces
\[
Q_{b,t,:}=H_{b,t,:}W_Q
\]
usa el mismo \(W_Q\) para todo \(t\). Esto es peso compartido a traves de la
dimension temporal. Reduce parametros y, mas importante, impone equivarianza
estructural: el mismo patron local puede reconocerse en distintas posiciones.
\end{remark}

\begin{remark}[Esparsidad estructural por mascara causal]
La mascara causal no es esparsidad de parametros, sino esparsidad de
conectividad informacional. Obliga
\[
\alpha_{t,s}=0\qquad\text{si }s>t,
\]
de modo que la representacion de la posicion \(t\) no use informacion futura.
La restriccion no reduce directamente \(\mathrm{size}(f_\theta)\), pero reduce
la clase de dependencias funcionales permitidas.
\end{remark}

\subsection{Activaciones y flujo de gradiente}

\begin{definition}[Sigmoide, ReLU, SiLU y GELU]
Las activaciones discutidas en \code{capitulo02 (1).tex} incluyen
\[
\sigma_{\mathrm{sig}}(x)=\frac{1}{1+e^{-x}},
\qquad
\sigma_{\mathrm{ReLU}}(x)=\max\{x,0\},
\]
\[
\SiLU(x)=x\sigma_{\mathrm{sig}}(x),
\qquad
\GELU(x)=x\Phi_{\mathcal N(0,1)}(x),
\]
donde \(\Phi_{\mathcal N(0,1)}\) es la funcion de distribucion normal estandar.
\end{definition}

\begin{remark}[Gradientes desvanecientes y explosivos]
Si una composicion profunda satisface, idealizadamente,
\[
\sup_x|\sigma'(x)|\leq 1-\delta,
\]
entonces los productos de derivadas pueden decrecer como \((1-\delta)^L\). Si
las derivadas o normas Jacobianas son mayores que uno, pueden crecer
exponencialmente. Este es el problema de gradiente desvaneciente o explosivo.
En el repo, los residuales, LayerNorm/RMSNorm, RoPE y activaciones suaves como
\(\GELU\) o \(\SiLU\) no eliminan toda dificultad de optimizacion, pero ayudan a
mantener una geometria entrenable frente a composiciones profundas.
\end{remark}

\begin{remark}[Por que SwiGLU usa SiLU]
SwiGLU usa
\[
\SiLU(W_gx)\odot W_ux
\]
como compuerta suave. Frente a una ReLU pura, SiLU evita una discontinuidad de la
derivada en cero y conserva una senal suave de gradiente. Frente a una sigmoide
pura, multiplica por \(x\), lo que permite respuestas no acotadas hacia valores
positivos grandes. Esta es una de las razones por las que la variante LLaMA-style
del repo no es solo un Transformer con nombres cambiados.
\end{remark}

\subsection{Softmax, NLL y entropia cruzada}

\begin{definition}[Softmax]
Para \(z\in\R^m\),
\[
\softmax(z)_j=\frac{e^{z_j}}{\sum_{r=0}^{m-1}e^{z_r}}.
\]
\end{definition}

\begin{definition}[Negative log-likelihood categorica]
Si la clase correcta es \(y\), la perdida es
\[
\ell(z,y)=-\log\softmax(z)_y.
\]
\end{definition}

\begin{proposition}
Minimizar la entropia cruzada empirica equivale a maximizar la log-verosimilitud
del token observado bajo un modelo categorico.
\end{proposition}

\begin{proof}
Para observaciones \(y_i\) con logits \(z_i\),
\[
\prod_i \softmax(z_i)_{y_i}
\]
es la verosimilitud. Su logaritmo negativo es
\[
-\sum_i\log\softmax(z_i)_{y_i},
\]
que coincide con la suma de perdidas de entropia cruzada.
\end{proof}

\subsection{Expresividad no es aprendibilidad}

Los resultados de aproximacion universal, discutidos por Petersen--Zech,
Calin y Wegner, explican que redes suficientemente grandes pueden aproximar
clases amplias de funciones. Eso no implica que el entrenamiento de este repo
encuentre la funcion ideal. En la practica intervienen:
\[
\text{arquitectura},\quad
\text{parametros},\quad
\text{datos},\quad
\text{optimizador},\quad
\text{pasos},\quad
\text{ruido de minibatch}.
\]
Esta distincion es central para interpretar que el corpus high sea mas diverso
pero produzca peor test loss bajo presupuesto fijo.

\section{Tokenizacion y datasets en el repo}

\subsection{\texttt{CharTokenizer}}

\begin{definition}[Vocabulario finito]
Para un texto \(c_0\cdots c_{N-1}\), el vocabulario character-level es
\[
C=\{c_t:0\le t<N\}.
\]
Si \(|C|=m\), \code{CharTokenizer.from\_text} construye una biyeccion
\[
\tau:C\to\{0,\ldots,m-1\}.
\]
\end{definition}

\begin{definition}[Codificacion y decodificacion]
La codificacion es
\[
\mathrm{encode}(c_0\cdots c_{N-1})
=
(\tau(c_0),\ldots,\tau(c_{N-1})).
\]
La decodificacion aplica \(\tau^{-1}\) token por token.
\end{definition}

\begin{remark}[Limitaciones]
El tokenizador no aprende morfemas ni subpalabras. Esto hace el experimento
limpio matematicamente, porque el vocabulario es exactamente el conjunto de
caracteres observados. Pero tambien limita la semantica: el modelo aprende
regularidades de caracteres, no unidades lexicas de alto nivel.
\end{remark}

\subsection{\texttt{TokenDataset}}

\begin{definition}
\code{TokenDataset(tokens, block\_size=B)} implementa
\[
i\mapsto (X_i,Y_i)
\]
con
\[
X_i=(x_i,\ldots,x_{i+B-1}),\qquad
Y_i=(x_{i+1},\ldots,x_{i+B}).
\]
\end{definition}

\begin{remark}[Funcion de perdida por posicion]
Aunque \(Y_i\) es una secuencia, la perdida se calcula por posicion:
\[
\ell_i(\theta)=\frac1B\sum_{t=0}^{B-1}
-\log p_\theta(Y_{i,t}\mid X_{i,\le t}).
\]
La mascara causal garantiza que la posicion \(t\) no use \(X_{i,t+1}\) ni
tokens futuros.
\end{remark}

\section{Modelo bigrama}

\subsection{Definicion matematica}

\begin{definition}[Bigrama]
Sea \(m=|\calV|\). El modelo bigrama aprende una matriz
\[
A\in\R^{m\times m}.
\]
Para un token actual \(i\), la distribucion del siguiente token es
\[
p_\theta(x_{t+1}=j\mid x_t=i)=\softmax(A_{i,:})_j.
\]
\end{definition}

\begin{remark}[Codigo]
En \code{BigramLanguageModel}, \code{nn.Embedding(vocab\_size, vocab\_size)}
es precisamente una tabla de \(m\) filas y \(m\) columnas. Consultar el token
\(i\) devuelve la fila \(A_{i,:}\).
\end{remark}

\subsection{Capacidad y limitaciones}

\begin{proposition}
El bigrama no puede distinguir dos contextos con el mismo ultimo token.
\end{proposition}

\begin{proof}
Por construccion,
\[
p_\theta(x_{t+1}\mid x_0,\ldots,x_t)=p_\theta(x_{t+1}\mid x_t).
\]
Si dos historias terminan en el mismo \(x_t=i\), ambas seleccionan la misma
fila \(A_{i,:}\).
\end{proof}

\begin{remark}[Relacion con entropia condicional]
El mejor bigrama posible, en el limite de datos suficientes, se aproxima a la
distribucion empirica
\[
\widehat p(j\mid i)=
\frac{\#\{t:x_t=i,x_{t+1}=j\}}{\#\{t:x_t=i\}}.
\]
La dificultad residual de este modelo esta relacionada con
\[
H(X_{t+1}\mid X_t).
\]
Si esa entropia es alta, conocer solo el caracter actual deja mucha
incertidumbre.
\end{remark}

\section{Atencion causal}

\subsection{Tensores de entrada}

En \code{CausalSelfAttentionHead}, la entrada tiene forma
\[
X\in\R^{M\times T\times d},
\]
donde \(M\) es batch size, \(T\le B\) es longitud de secuencia y \(d\) es
\code{embedding\_dim}. Cada posicion \(t\) tiene un vector \(x_t\in\R^d\).

\subsection{Queries, keys, values}

\begin{definition}
Una cabeza de atencion aprende matrices
\[
W_Q,W_K,W_V\in\R^{d\times d_h},
\]
y define
\[
Q=XW_Q,\qquad K=XW_K,\qquad V=XW_V.
\]
Por batch,
\[
Q,K,V\in\R^{T\times d_h}.
\]
\end{definition}

\begin{definition}[Scores escalados]
\[
S=\frac{QK^\top}{\sqrt{d_h}}\in\R^{T\times T}.
\]
La entrada \(S_{ts}\) mide compatibilidad entre la query de la posicion \(t\)
y la key de la posicion \(s\).
\end{definition}

\begin{remark}
El factor \(1/\sqrt{d_h}\) estabiliza la escala de los productos punto. Sin el,
si las coordenadas tienen varianza comparable, el producto punto crece con
\(d_h\) y puede saturar el softmax.
\end{remark}

\subsection{Mascara causal y softmax}

\begin{definition}[Mascara causal]
La mascara \(M\in\{0,-\infty\}^{T\times T}\) cumple
\[
M_{ts}=
\begin{cases}
0,&s\le t,\\
-\infty,&s>t.
\end{cases}
\]
Entonces
\[
\alpha_{t,:}=\softmax(S_{t,:}+M_{t,:})
\]
asigna masa solo a posiciones pasadas o presentes.
\end{definition}

\begin{definition}[Salida]
\[
O=\alpha V,\qquad
O_t=\sum_{s\le t}\alpha_{ts}V_s.
\]
\end{definition}

\begin{remark}[Codigo]
\code{causal\_mask(block\_size)} crea una matriz triangular inferior booleana.
Luego \code{scores.masked\_fill(\textasciitilde mask, -inf)} implementa la
mascara \(M\).
\end{remark}

\section{Transformer pequeño}

\subsection{Embeddings}

\begin{definition}[Embedding de tokens]
\code{nn.Embedding(vocab\_size, embedding\_dim)} aprende una matriz
\[
E\in\R^{m\times d}.
\]
El token \(i\) se representa como \(E_i\in\R^d\).
\end{definition}

\begin{definition}[Embedding posicional absoluto]
El Transformer pequeno aprende
\[
P\in\R^{B\times d}.
\]
Para una secuencia \(x_0,\ldots,x_{T-1}\), la entrada al primer bloque es
\[
H^{(0)}_t=E_{x_t}+P_t.
\]
\end{definition}

\subsection{Bloque pre-norm}

\begin{definition}[LayerNorm idealizada]
Para \(x\in\R^d\),
\[
\mathrm{LN}(x)=\gamma\odot\frac{x-\mu(x)\mathbf 1}{\sqrt{\sigma^2(x)+\epsilon}}+\beta,
\]
donde
\[
\mu(x)=\frac1d\sum_{j=1}^dx_j,\qquad
\sigma^2(x)=\frac1d\sum_{j=1}^d(x_j-\mu(x))^2.
\]
\end{definition}

\begin{definition}[Bloque Transformer del repo]
El bloque de \code{TransformerBlock} es
\[
\widetilde H^{(\ell)}
=
H^{(\ell)}+\mathrm{MHA}(\mathrm{LN}_1(H^{(\ell)})),
\]
\[
H^{(\ell+1)}
=
\widetilde H^{(\ell)}
+
\mathrm{FFN}(\mathrm{LN}_2(\widetilde H^{(\ell)})).
\]
\end{definition}

\begin{definition}[Feedforward GELU]
La red feedforward usada es
\[
\mathrm{FFN}(x)
=
W_2\,\GELU(W_1x+b_1)+b_2,
\]
con dimension oculta \(4d\).
\end{definition}

\subsection{Cabeza de lenguaje}

Tras \(L\) bloques,
\[
H^{(L)}\in\R^{M\times T\times d}.
\]
La capa final produce logits
\[
Z=H^{(L)}W_{\mathrm{lm}}+b_{\mathrm{lm}}
\in\R^{M\times T\times m}.
\]
El modelo estima
\[
p_\theta(x_{t+1}=j\mid x_{\le t})
=
\softmax(Z_{t,:})_j.
\]

\section{Modelo LLaMA-style}

El modelo \code{LlamaStyleLanguageModel} conserva el objetivo autoregresivo,
pero reemplaza tres componentes del Transformer pequeno.

\subsection{RMSNorm}

\begin{definition}
Para \(x\in\R^d\),
\[
\RMS(x)=\sqrt{\frac1d\sum_{j=1}^d x_j^2+\epsilon}.
\]
RMSNorm es
\[
\mathrm{RMSNorm}(x)
=
g\odot \frac{x}{\RMS(x)}.
\]
\end{definition}

\begin{remark}
A diferencia de LayerNorm, RMSNorm no resta la media. En el codigo,
\code{variance = x.pow(2).mean(dim=-1, keepdim=True)} y
\code{x * torch.rsqrt(variance + eps)} implementan exactamente esta formula.
\end{remark}

\subsection{RoPE}

\begin{definition}[Rotacion por pares]
Sea \(x\in\R^{d_h}\) con \(d_h\) par. Para cada par
\((x_{2r},x_{2r+1})\), RoPE aplica
\[
\begin{pmatrix}
x'_{2r}\\x'_{2r+1}
\end{pmatrix}
=
\begin{pmatrix}
\cos\alpha_{t,r}&-\sin\alpha_{t,r}\\
\sin\alpha_{t,r}&\cos\alpha_{t,r}
\end{pmatrix}
\begin{pmatrix}
x_{2r}\\x_{2r+1}
\end{pmatrix},
\]
donde
\[
\alpha_{t,r}=t\cdot 10000^{-2r/d_h}.
\]
\end{definition}

\begin{remark}[Posicion relativa]
El producto punto entre queries y keys rotadas depende de diferencias de
posicion a traves de identidades trigonometricas. Por eso RoPE codifica
informacion posicional sin sumar un embedding absoluto \(P_t\). En el codigo,
\code{apply\_rope} se aplica a \(q\) y \(k\), no a \(v\).
\end{remark}

\subsection{SwiGLU}

\begin{definition}
\[
\SiLU(u)=u\,\sigma(u),\qquad \sigma(u)=\frac1{1+e^{-u}}.
\]
El bloque SwiGLU implementa
\[
\mathrm{SwiGLU}(x)
=
W_d\left(\SiLU(W_gx)\odot W_ux\right).
\]
\end{definition}

\begin{remark}
La multiplicacion elemento a elemento crea una compuerta aprendible. Una parte
del feedforward regula cuanta informacion pasa por la otra. Esto reemplaza el
MLP GELU del Transformer pequeno.
\end{remark}

\subsection{Atencion LLaMA-style}

La atencion LLaMA-style usa proyecciones sin sesgo:
\[
q=xW_Q,\qquad k=xW_K,\qquad v=xW_V,
\]
aplica RoPE a \(q,k\), calcula
\[
\softmax\left(\frac{qk^\top}{\sqrt{d_h}}+M\right)v,
\]
y proyecta con \(W_O\), tambien sin sesgo. La estructura residual es
\[
\widetilde H=H+\mathrm{Attn}(\mathrm{RMSNorm}(H)),
\]
\[
H^+=\widetilde H+\mathrm{SwiGLU}(\mathrm{RMSNorm}(\widetilde H)).
\]

\section{Entrenamiento, evaluacion y protocolo experimental}

\subsection{\texttt{scripts/train.py}}

El script de entrenamiento realiza:
\begin{enumerate}
\item lee texto UTF-8;
\item construye \(\tau\) con \code{CharTokenizer.from\_text};
\item codifica \(x_0,\ldots,x_{N-1}\);
\item divide train/val/test;
\item construye \code{TokenDataset} para train y val;
\item construye el modelo seleccionado;
\item minimiza entropia cruzada con AdamW;
\item guarda checkpoint con parametros, tokenizer y perdidas finales.
\end{enumerate}

Matematicamente, cada paso de entrenamiento toma un batch
\[
(X,Y)\in\calV^{M\times B}\times\calV^{M\times B},
\]
calcula
\[
Z_\theta=f_\theta(X)\in\R^{M\times B\times m},
\]
y minimiza
\[
\widehat R_B(\theta)=
\frac1{MB}\sum_{a,t}-\log\softmax(Z_{\theta,a,t,:})_{Y_{a,t}}.
\]

\subsection{\texttt{scripts/evaluate.py}}

La evaluacion reconstruye el modelo desde el checkpoint y estima perdida en el
split de test. Si \(\ell_1,\ldots,\ell_K\) son perdidas de batches aleatorios,
reporta
\[
\bar\ell=\frac1K\sum_{k=1}^K\ell_k,
\]
\[
s=\sqrt{\frac1{K-1}\sum_{k=1}^K(\ell_k-\bar\ell)^2},
\qquad
\mathrm{se}=\frac{s}{\sqrt K}.
\]
Tambien reporta
\[
\PPL=\exp(\bar\ell),
\qquad
\mathrm{gap}=\bar\ell-\mathrm{train\ loss}.
\]

\begin{remark}
El intervalo \(\bar\ell\pm1.96\,\mathrm{se}\) en el script es descriptivo.
Debido a ventanas solapadas y batches extraidos de una secuencia dependiente,
no debe presentarse como intervalo de confianza iid estricto.
\end{remark}

\subsection{\texttt{scripts/run\_effective\_tokens.py}}

El protocolo de tokens efectivos hace tres cosas matematicamente importantes:
\begin{enumerate}
\item fuerza igual tamano bruto de corpus, \(N_{\mathrm{chars}}\), mediante
truncamiento o validacion;
\item calcula metricas de diversidad sobre cada corpus;
\item entrena/evalua los mismos modelos bajo hiperparametros comparables.
\end{enumerate}
Asi, la variable experimental principal no es el tamano bruto, sino la
estructura estadistica del texto.

\section{Metricas de corpus y tokens efectivos}

\subsection{Entropia empirica}

\begin{definition}[Distribucion empirica de n-gramas]
Para un texto \(c_0,\ldots,c_{N-1}\), el conteo del n-grama
\(g=(a_1,\ldots,a_n)\) es
\[
C_n(g)=\#\{t:(c_t,\ldots,c_{t+n-1})=g\}.
\]
La frecuencia empirica es
\[
\widehat p_n(g)=\frac{C_n(g)}{N-n+1}.
\]
\end{definition}

\begin{definition}[Entropia empirica de n-gramas]
\[
\widehat H_n
=
-\sum_g\widehat p_n(g)\log\widehat p_n(g).
\]
\end{definition}

\begin{remark}
\code{character\_entropy} calcula \(\widehat H_1\);
\code{ngram\_entropy(text,n)} calcula \(\widehat H_n\). Estas entropias son
locales: no capturan directamente coherencia semantica global.
\end{remark}

\subsection{Entropia condicional de bigramas}

\begin{definition}
\[
\widehat H(X_{t+1}\mid X_t)
=
\widehat H(X_t,X_{t+1})-\widehat H(X_t).
\]
\end{definition}

\begin{remark}
Esta metrica se conecta con el bigrama: si la entropia condicional es alta, un
modelo que solo conoce \(X_t\) enfrenta alta incertidumbre residual.
\end{remark}

\subsection{Distinct-n y gzip}

\begin{definition}[Distinct-n]
\[
\mathrm{distinct}_n=
\frac{\#\{\text{n-gramas distintos}\}}{\#\{\text{n-gramas totales}\}}.
\]
\end{definition}

\begin{definition}[Gzip ratio]
El repo define
\[
\mathrm{gzip\ ratio}
=
\frac{\text{tamano comprimido}}{\text{tamano original}}.
\]
Valores mayores indican menor compresibilidad bajo gzip.
\end{definition}

\begin{remark}
Gzip mide redundancia explotable por un compresor especifico. No mide calidad
literaria, factualidad ni utilidad semantica. Por eso no debe confundirse con
la nocion completa de calidad de datos de Chang et al.
\end{remark}

\subsection{Tokens efectivos}

Chang et al. proponen que el numero bruto de tokens no basta para predecir
rendimiento; la calidad de datos modifica la cantidad de informacion util para
el modelo. En su trabajo aparecen senales como diversidad y syntheticity. Este
repo no estima syntheticity con un modelo docente. Aisla una dimension:
\[
\text{diversidad textual empirica}.
\]
La hipotesis no es que diversidad sea identica a tokens efectivos, sino que
podria funcionar como proxy parcial si el modelo puede convertir variacion en
estructura predictiva.

\section{Interpretacion matematica del resultado}

El corpus high tiene mayor vocabulario, mayor \(\widehat H_n\), mayor
distinct-4 y mayor gzip ratio. Por tanto, es mas diverso segun metricas locales
y de compresibilidad. Sin embargo, los modelos pequenos producen mayor test
loss en high que en medium.

La interpretacion rigurosa es:
\[
\text{mayor diversidad empirica}
\not\Rightarrow
\text{menor riesgo de test bajo presupuesto fijo}.
\]
La diversidad aumenta el conjunto de patrones posibles. Si la capacidad, el
numero de pasos o el sesgo arquitectural no bastan, esa riqueza se manifiesta
como mayor dificultad predictiva.

\begin{remark}[Punto central]
El resultado negativo no contradice la idea de tokens efectivos. La refina:
un token es efectivo solo relativo a un sistema completo
\[
(\text{datos},\text{modelo},\text{optimizador},\text{presupuesto}).
\]
\end{remark}

\section{Auditoria matematica modulo por modulo}

Esta seccion escribe, de manera deliberadamente redundante, la correspondencia
entre cada modulo del repo y el objeto matematico que implementa. La redundancia
es util: el lector puede leer esta parte como una guia de traduccion entre
PyTorch y notacion matematica.

\subsection{\code{tokenizer.py}: vocabulario finito y codificacion}

\begin{definition}[Vocabulario observado]
Sea \(s=(c_0,\dots,c_{N-1})\) el texto de entrenamiento, visto como una
secuencia de caracteres. El tokenizer define
\[
\calV=\{c\colon c=c_t\text{ para algun }0\leq t<N\},
\qquad
m=|\calV|.
\]
Luego ordena \(\calV\) y construye dos mapas inversos
\[
\phi:\calV\to\{0,\dots,m-1\},\qquad
\psi:\{0,\dots,m-1\}\to\calV,
\]
con \(\psi(\phi(c))=c\) para todo \(c\in\calV\).
\end{definition}

\begin{remark}[Entrada, salida y limitacion]
La entrada de \code{encode} es una cadena \(s\in\calV^N\) y su salida es
\[
x=(\phi(c_0),\dots,\phi(c_{N-1}))\in\{0,\dots,m-1\}^N.
\]
La entrada de \code{decode} es una secuencia de indices y la salida es la cadena
obtenida aplicando \(\psi\) coordenada a coordenada. Como no hay token
\emph{unknown}, el modelo queda definido sobre el vocabulario observado. Esto
es matematicamente limpio, pero restringe la extrapolacion: un caracter no visto
no pertenece al espacio muestral tokenizado del experimento.
\end{remark}

\begin{example}[Por que character-level no es neutro]
Si el texto contiene las palabras ``transformer'' y ``transformacion'', el
tokenizer de caracteres comparte subcadenas a nivel de letras, pero no crea una
unidad semantica para prefijos, raices o morfemas. Por tanto, el modelo estima
una distribucion sobre \(\calV^\N\), no directamente sobre palabras ni conceptos.
Esta eleccion reduce el tamano del vocabulario, pero aumenta la longitud de las
dependencias necesarias para representar patrones lexicos.
\end{example}

\subsection{\code{data.py}: ventanas solapadas y dependencia}

El dataset transforma la secuencia tokenizada \(x_0,\dots,x_{N-1}\) en pares
\[
X_i=(x_i,\dots,x_{i+B-1})\in\{0,\dots,m-1\}^{B},
\quad
Y_i=(x_{i+1},\dots,x_{i+B})\in\{0,\dots,m-1\}^{B},
\]
para \(0\leq i\leq N-B-1\). Aqui \(B=\code{block\_size}\). La salida de
\code{\_\_getitem\_\_} son tensores de forma \((B,)\), y
\code{batch\_to\_tensors} los apila en
\[
X,Y\in\{0,\dots,m-1\}^{M\times B},
\]
donde \(M=\code{batch\_size}\).

\begin{proposition}[Las ventanas no son iid]
Para \(i\) y \(i+1\), las ventanas comparten \(B-1\) coordenadas:
\[
X_i=(x_i,x_{i+1},\dots,x_{i+B-1}),\qquad
X_{i+1}=(x_{i+1},\dots,x_{i+B}).
\]
Por tanto, \(X_i\) y \(X_{i+1}\) no son independientes salvo en casos
degenerados. Mas aun, las etiquetas tambien se solapan:
\[
Y_i=(x_{i+1},\dots,x_{i+B}),\qquad
Y_{i+1}=(x_{i+2},\dots,x_{i+B+1}).
\]
\end{proposition}

\begin{remark}[Consecuencia para aprendizaje estadistico]
La teoria ERM clasica suele formularse con muestras iid
\((Z_i)_{i=1}^n\sim\calD^n\). El repo usa una aproximacion empirica distinta:
ventanas extraidas de una unica realizacion textual larga. La perdida promedio
sigue siendo una cantidad observable y optimizable, pero las cotas iid deben
interpretarse como heuristica o reemplazarse por argumentos para procesos
dependientes. Esta es una diferencia matematica importante entre el experimento
real y la formulacion idealizada de aprendizaje estadistico.
\end{remark}

\subsection{\code{corpus\_stats.py}: estadisticos de informacion}

El archivo \code{corpus\_stats.py} no aprende parametros. Define funciones
\[
S:\calV^N\to\R^k
\]
que resumen propiedades empiricas del texto. Para caracteres,
\[
\widehat p(a)=\frac{1}{N}\sum_{t=0}^{N-1}\mathbf 1\{x_t=a\}.
\]
La entropia empirica es
\[
\widehat H_1=-\sum_{a\in\calV}\widehat p(a)\log_2\widehat p(a).
\]
Para n-gramas \(u=(u_1,\dots,u_n)\),
\[
\widehat p_n(u)=
\frac{1}{N-n+1}\sum_{t=0}^{N-n}\mathbf 1\{(x_t,\dots,x_{t+n-1})=u\},
\]
y
\[
\widehat H_n=-\sum_{u\in\calV^n}\widehat p_n(u)\log_2\widehat p_n(u).
\]

\begin{remark}[Que mide y que no mide]
\(\widehat H_n\) mide dispersion empirica de patrones locales. No mide
veracidad, coherencia discursiva, dificultad semantica ni utilidad para una
arquitectura concreta. \(\mathrm{distinct}_n\) mide soporte relativo observado,
mientras que gzip ratio mide compresibilidad por un algoritmo especifico. En la
terminologia motivada por Chang et al., estas metricas son proxies parciales de
calidad/diversidad, no una medicion completa de tokens efectivos.
\end{remark}

\subsection{\code{bigram.py}: matriz de transicion aprendida}

El modelo bigrama contiene una tabla de embeddings
\[
A\in\R^{m\times m}.
\]
Aunque PyTorch la llama \code{token\_embedding\_table}, matematicamente cada fila
es un vector de logits:
\[
A_{i,:}=(A_{i0},\dots,A_{i,m-1}).
\]
Para una entrada \(X\in\{0,\dots,m-1\}^{M\times T}\), el modelo produce
\[
Z_{b,t,:}=A_{X_{b,t},:}\in\R^m.
\]
Por tanto, los logits tienen dimension
\[
Z\in\R^{M\times T\times m}.
\]

\begin{definition}[Distribucion bigrama]
La probabilidad asignada al siguiente token es
\[
p_\theta(Y_{b,t}=j\mid X_{b,t}=i)
=
\frac{\exp(A_{ij})}{\sum_{\ell=0}^{m-1}\exp(A_{i\ell})}.
\]
\end{definition}

\begin{proposition}[Capacidad del bigrama]
El bigrama puede representar exactamente cualquier cadena de Markov homogenea
de orden uno sobre \(\calV\) cuyas probabilidades de transicion sean positivas.
No puede representar dependencias que requieran distinguir dos historias con el
mismo ultimo caracter.
\end{proposition}

\begin{proof}
Dada una matriz de transicion \(P\) con \(P_{ij}>0\), basta tomar
\(A_{ij}=\log P_{ij}\). Como cada fila suma uno, \(\softmax(A_{i,:})=P_{i,:}\).
Si dos historias \(h\) y \(h'\) terminan en el mismo token \(i\), el modelo usa
la misma fila \(A_{i,:}\), luego asigna la misma distribucion condicional aunque
la distribucion real dependa de tokens anteriores.
\end{proof}

\subsection{\code{attention.py}: atencion como mezcla adaptativa}

Para una cabeza, la entrada es
\[
X\in\R^{M\times T\times d}.
\]
Los parametros aprendidos son
\[
W_Q,W_K,W_V\in\R^{d\times d_h}
\]
sin bias. Las proyecciones son
\[
Q=XW_Q,\qquad K=XW_K,\qquad V=XW_V,
\]
con \(Q,K,V\in\R^{M\times T\times d_h}\). Para cada batch \(b\), el score entre
posiciones \(t\) y \(s\) es
\[
S^{(b)}_{ts}=
\frac{\langle Q_{b,t,:},K_{b,s,:}\rangle}{\sqrt{d_h}}.
\]
La mascara causal reemplaza por \(-\infty\) los scores con \(s>t\):
\[
\widetilde S^{(b)}_{ts}
=
\begin{cases}
S^{(b)}_{ts}, & s\leq t,\\
-\infty, & s>t.
\end{cases}
\]
Los pesos son
\[
\alpha^{(b)}_{ts}
=
\frac{\exp(\widetilde S^{(b)}_{ts})}
{\sum_{r=0}^{t}\exp(\widetilde S^{(b)}_{tr})},
\qquad
\sum_{s=0}^{t}\alpha^{(b)}_{ts}=1,
\]
y la salida
\[
O_{b,t,:}=\sum_{s=0}^{t}\alpha^{(b)}_{ts}V_{b,s,:}.
\]

\begin{remark}[Interpretacion]
Cada vector de salida es una combinacion convexa de valores pasados y presentes.
La combinacion depende de la entrada porque los pesos \(\alpha_{ts}\) dependen
de \(Q\) y \(K\). Esta es la diferencia central con un filtro lineal fijo: la
matriz de mezcla se recalcula para cada secuencia.
\end{remark}

\subsection{\code{causal\_mask.py}: estructura triangular}

La mascara es la matriz booleana
\[
C\in\{0,1\}^{B\times B},
\qquad
C_{ts}=\mathbf 1\{s\leq t\}.
\]
En el codigo se registra como buffer, no como parametro. Esto significa que no
aparece en \(\theta\) ni recibe gradiente. Su funcion es imponer una restriccion
estructural sobre la clase de funciones:
\[
f_\theta(x_0,\dots,x_{T-1})_t
\text{ depende solo de }x_0,\dots,x_t.
\]
Sin esta restriccion, el entrenamiento de next-token prediction podria mirar
tokens futuros y la perdida dejaria de medir prediccion autoregresiva.

\subsection{\code{transformer.py}: composicion residual pre-norm}

El Transformer pequeno usa embeddings de token y posicion:
\[
E\in\R^{m\times d},\qquad P\in\R^{B\times d}.
\]
Para \(X\in\{0,\dots,m-1\}^{M\times T}\),
\[
H^{(0)}_{b,t,:}=E_{X_{b,t},:}+P_{t,:}.
\]
Cada bloque aplica
\[
\widetilde H^{(\ell)}
=
H^{(\ell)}+\mathrm{MHA}(\mathrm{LN}(H^{(\ell)})),
\]
\[
H^{(\ell+1)}
=
\widetilde H^{(\ell)}
+\mathrm{FFN}(\mathrm{LN}(\widetilde H^{(\ell)})).
\]
La cabeza de lenguaje es una transformacion afin
\[
Z_{b,t,:}=W_{\mathrm{lm}}H^{(L)}_{b,t,:}+b_{\mathrm{lm}},
\qquad
Z\in\R^{M\times T\times m}.
\]

\begin{remark}[Papel de los residuales]
Los residuales hacen que cada bloque aprenda una correccion de la representacion,
no una representacion completamente nueva. Desde el punto de vista de
optimizacion, esto mejora el flujo de gradiente porque existe una ruta directa
entre capas. En terminos funcionales, el bloque tiene forma \(x\mapsto x+g(x)\).
\end{remark}

\subsection{\code{rmsnorm.py}: normalizacion por escala}

Para \(x\in\R^d\), el modulo implementa
\[
\mathrm{RMSNorm}(x)
=
g\odot
\frac{x}{\sqrt{d^{-1}\sum_{j=1}^d x_j^2+\epsilon}},
\qquad
g\in\R^d.
\]
La entrada y salida tienen la misma dimension. El unico parametro aprendido es
\(g\). A diferencia de LayerNorm, no se resta la media; por tanto RMSNorm
controla la escala cuadratica media, no centra el vector.

\begin{remark}[Efecto en el modelo LLaMA-style]
RMSNorm aparece antes de atencion y feedforward:
\[
x\mapsto x+\mathrm{Attn}(\mathrm{RMSNorm}(x)),
\qquad
x\mapsto x+\mathrm{SwiGLU}(\mathrm{RMSNorm}(x)).
\]
Asi, los submodulos reciben vectores reescalados, mientras la ruta residual
preserva la magnitud acumulada de la representacion.
\end{remark}

\subsection{\code{rope.py}: posicion como rotacion}

RoPE actua sobre pares de coordenadas. Si
\[
u=(u_0,u_1,\dots,u_{d_h-2},u_{d_h-1}),
\]
se agrupan pares \((u_{2r},u_{2r+1})\). Para una posicion \(t\), se define una
matriz de rotacion por bloque
\[
R_t=\diag(R(t\omega_0),R(t\omega_1),\dots,R(t\omega_{d_h/2-1})),
\]
donde
\[
R(\alpha)=
\begin{pmatrix}
\cos\alpha & -\sin\alpha\\
\sin\alpha & \cos\alpha
\end{pmatrix}.
\]
El codigo aplica
\[
q_t\mapsto R_t q_t,\qquad k_s\mapsto R_s k_s.
\]
Entonces el producto usado por atencion es
\[
\langle R_t q_t,R_s k_s\rangle
=
q_t^\top R_t^\top R_s k_s
=
q_t^\top R_{s-t} k_s,
\]
porque las rotaciones por angulos se componen restando angulos. Esta identidad
es la razon matematica de que RoPE codifique informacion relativa \(s-t\) dentro
del producto query-key.

\subsection{\code{swiglu.py}: compuerta multiplicativa}

Para \(x\in\R^d\), el feedforward LLaMA-style no es simplemente
\(\sigma(Wx)\). Usa dos proyecciones internas:
\[
u=W_u x,\qquad g=W_g x,\qquad u,g\in\R^h.
\]
Luego calcula
\[
\mathrm{SwiGLU}(x)
=
W_d\left(\SiLU(g)\odot u\right),
\qquad
\SiLU(a)=a\sigma(a).
\]
En tensores, si \(X\in\R^{M\times T\times d}\), entonces la salida tiene forma
\(\R^{M\times T\times d}\). Los parametros son
\[
W_u,W_g\in\R^{d\times h},\qquad W_d\in\R^{h\times d},
\]
mas sesgos si el modulo local los define. El producto coordenada a coordenada
crea una compuerta dependiente de la entrada: algunas coordenadas internas son
amplificadas o atenuadas segun \(\SiLU(W_gx)\).

\subsection{\code{llama.py}: comparacion estructural con el Transformer pequeno}

El modelo LLaMA-style conserva el objetivo autoregresivo y las formas de entrada
y salida:
\[
X,Y\in\{0,\dots,m-1\}^{M\times T},
\qquad
Z_\theta(X)\in\R^{M\times T\times m}.
\]
Pero cambia la parametrizacion interna:
\[
\begin{array}{lll}
\text{Transformer pequeno} & \text{LLaMA-style} & \text{efecto matematico}\\
\hline
\text{positional embedding }P_t
& \text{RoPE en }Q,K
& \text{posicion absoluta vs relativa en scores}\\
\text{LayerNorm}
& \text{RMSNorm}
& \text{centrado+escala vs solo escala}\\
\text{FFN con activacion simple}
& \text{SwiGLU}
& \text{no linealidad con compuerta}\\
\text{proyecciones con bias}
& \text{varias proyecciones sin bias}
& \text{menor parametrizacion afin}\\
\end{array}
\]
La diferencia no es cosmetica: define otra clase de hipotesis
\(\calH_{\mathrm{llama}}\), con sesgos inductivos distintos para posicion,
escala y mezcla no lineal.

\subsection{\code{train.py}: problema de optimizacion real}

El script elige un modelo \(f_\theta\), un corpus, un tokenizer y una particion.
La funcion que se minimiza en cada minibatch es
\[
\ell_M(\theta)=
\frac{1}{MT}\sum_{b=1}^{M}\sum_{t=1}^{T}
-\log
\frac{\exp Z_{\theta,b,t,Y_{b,t}}}
{\sum_{j=0}^{m-1}\exp Z_{\theta,b,t,j}}.
\]
La actualizacion AdamW puede verse como una transformacion
\[
\theta_{k+1}=\mathcal A_k(\theta_k,\nabla\ell_M(\theta_k)),
\]
donde \(\mathcal A_k\) mantiene momentos de primer y segundo orden y aplica
decaimiento de peso desacoplado. El resultado del entrenamiento no es solo el
minimo de una funcion abstracta; depende de la semilla, el orden de minibatches,
el learning rate, el numero de iteraciones y la arquitectura.

\subsection{\code{evaluate.py}: estimacion de riesgo de test}

La evaluacion repite \(\code{eval\_iters}\) minibatches de test y obtiene
perdidas \(L_1,\dots,L_r\). El script reporta
\[
\bar L=\frac1r\sum_{i=1}^r L_i,
\qquad
s^2=\frac1{r-1}\sum_{i=1}^r(L_i-\bar L)^2,
\qquad
\mathrm{SE}=\frac{s}{\sqrt r}.
\]
La perplejidad es
\[
\PPL=\exp(\bar L).
\]
Si la perdida esta en nats, \(\exp(\bar L)\) es el numero efectivo de opciones
uniformes equivalentes por prediccion. El gap reportado es
\[
\mathrm{gap}=\bar L_{\mathrm{test}}-\bar L_{\mathrm{train}}.
\]
Un gap positivo grande sugiere sobreajuste o desplazamiento entre particiones;
un gap pequeno con perdidas altas sugiere subajuste o dificultad intrinseca del
corpus para el modelo.

\subsection{\code{run\_effective\_tokens.py}: hipotesis experimental}

El protocolo compara corpus con presupuesto bruto controlado. Si dos corpus
tienen el mismo numero de caracteres/tokens brutos, pero difieren en diversidad,
el experimento pregunta si
\[
N_{\mathrm{efectivo}}(\text{high})
>
N_{\mathrm{efectivo}}(\text{medium})
\]
se manifiesta como menor perdida de test. La evidencia observada puede fallar
por tres razones matematicamente distintas:
\begin{enumerate}
\item La diversidad medida no coincide con informacion util para el objetivo
autoregresivo.
\item La informacion util existe, pero el modelo pequeno no tiene capacidad o
presupuesto para capturarla.
\item La diversidad aumenta tambien la entropia condicional, de modo que el
problema predictivo es mas dificil aunque el corpus sea mas rico.
\end{enumerate}

\subsection{\code{analyze\_corpus.py} y \code{summarize\_results.py}}

\code{analyze\_corpus.py} evalua \(S(s)\), el vector de estadisticos de corpus.
\code{summarize\_results.py} junta ese vector con metricas de entrenamiento:
\[
\left(S(s),\bar L_{\mathrm{train}},\bar L_{\mathrm{test}},
\PPL,\mathrm{gap}\right).
\]
Su papel matematico no es entrenar un modelo, sino construir una tabla para
comparar relaciones empiricas:
\[
\Delta S = S(s_{\mathrm{high}})-S(s_{\mathrm{medium}}),
\qquad
\Delta L = L_{\mathrm{high}}-L_{\mathrm{medium}}.
\]
El punto critico es no interpretar \(\Delta S>0\) como prueba automatica de
\(\Delta L<0\). La relacion entre diversidad y generalizacion esta mediada por
arquitectura, optimizacion y distribucion.

\section{Derivaciones adicionales}

\subsection{Entropia cruzada y gradiente respecto a logits}

Para un unico ejemplo con clase verdadera \(y\) y logits \(z\in\R^m\), la
perdida es
\[
\ell(z,y)
=
-\log\softmax(z)_y
=
-z_y+\log\sum_{j=0}^{m-1}\exp z_j.
\]
Derivando,
\[
\frac{\partial \ell}{\partial z_k}
=
-\mathbf 1\{k=y\}
+\frac{\exp z_k}{\sum_j\exp z_j}
=
\softmax(z)_k-\mathbf 1\{k=y\}.
\]
En el repo, esta derivada es la senal que PyTorch propaga desde
\code{F.cross\_entropy}. Para el bigrama, si \(z=A_{i,:}\), solo la fila
correspondiente al token actual recibe gradiente directo en ese ejemplo:
\[
\frac{\partial \ell}{\partial A_{ik}}
=
p_\theta(k\mid i)-\mathbf 1\{k=y\}.
\]
Esta formula muestra por que el bigrama aprende frecuencias condicionales:
empuja hacia arriba el logit del caracter observado y hacia abajo los demas en
proporcion a su probabilidad actual.

\subsection{Maxima verosimilitud de secuencias autoregresivas}

Un modelo autoregresivo factoriza
\[
p_\theta(x_0,\dots,x_{T})
=
p(x_0)\prod_{t=0}^{T-1}p_\theta(x_{t+1}\mid x_{\leq t}).
\]
El entrenamiento ignora el termino inicial y maximiza
\[
\sum_{t=0}^{T-1}\log p_\theta(x_{t+1}\mid x_{\leq t}).
\]
La perdida de next-token prediction es exactamente el negativo normalizado de
esta log-verosimilitud. En una ventana del repo, la historia disponible queda
truncada a longitud maxima \(B\):
\[
p_\theta(x_{t+1}\mid x_{\leq t})
\approx
p_\theta(x_{t+1}\mid x_{\max(0,t-B+1)},\dots,x_t).
\]
Por eso \(\code{block\_size}\) es un hiperparametro estadistico, no solo una
decision computacional: limita la memoria del estimador.

\subsection{Riesgo, aproximacion, estimacion y optimizacion}

Para una arquitectura fija \(\calH=\{f_\theta:\theta\in\Theta\}\), el exceso de
riesgo puede descomponerse informalmente como
\[
R(\widehat\theta)-R^\star
=
\underbrace{R(\theta^\star_{\calH})-R^\star}_{\text{error de aproximacion}}
+
\underbrace{R(\widehat\theta_{\mathrm{ERM}})-R(\theta^\star_{\calH})}_{\text{error de estimacion}}
+
\underbrace{R(\widehat\theta)-R(\widehat\theta_{\mathrm{ERM}})}_{\text{error de optimizacion}}.
\]
Esta descomposicion explica por que un corpus high puede dar peor perdida:
\begin{enumerate}
\item Si el modelo es demasiado pequeno, aumenta el error de aproximacion.
\item Si la muestra efectiva es baja por dependencia o redundancia, aumenta el
error de estimacion.
\item Si el paisaje no convexo es mas dificil o el presupuesto de entrenamiento
es insuficiente, aumenta el error de optimizacion.
\end{enumerate}

\subsection{Dimension efectiva y alta dimension}

Wegner enfatiza que muchos problemas de datos viven en espacios de alta
dimension pero poseen estructura explotable. En este repo, cada ventana
\[
X_i\in\{0,\dots,m-1\}^B
\]
tiene \(m^B\) configuraciones posibles. Incluso para \(m=80\) y \(B=128\),
ese espacio es astronomico. El aprendizaje solo es posible porque el texto no
ocupa uniformemente \(\calV^B\); tiene regularidades. Los modelos comparados
imponen distintas formas de compresion:
\[
\text{bigrama: } \calV^B\to\calV \text{ usando solo }x_t,
\]
\[
\text{Transformer: } \calV^B\to\R^d \text{ mediante atencion y composicion},
\]
\[
\text{LLaMA-style: } \calV^B\to\R^d \text{ con normalizacion, rotacion y gating}.
\]
Las metricas de corpus intentan medir parte de esa estructura, pero no la
dimension intrinseca completa del proceso textual.

\subsection{Compresion, SVD y embeddings}

Aunque el repo no calcula SVD durante entrenamiento, la idea de compresion lineal
ayuda a interpretar embeddings. La matriz de embeddings
\[
E\in\R^{m\times d}
\]
asigna a cada token un vector en dimension \(d\). Si se mirara una matriz de
coocurrencias \(C\in\R^{m\times m}\), una aproximacion de bajo rango
\[
C\approx U_r\Sigma_rV_r^\top
\]
representaria tokens por coordenadas latentes. El embedding aprendido cumple un
papel analogo, pero optimizado indirectamente por la perdida autoregresiva y no
por reconstruccion cuadratica. Esta conexion justifica incluir herramientas de
algebra lineal y compresion al explicar un modelo de lenguaje pequeno.

\subsection{Backpropagation como regla de la cadena organizada}

Sea el modelo una composicion
\[
f_\theta=f_L\circ f_{L-1}\circ\cdots\circ f_1.
\]
Si \(h_0=X\), \(h_\ell=f_\ell(h_{\ell-1};\theta_\ell)\) y
\(\ell=\mathcal L(h_L,Y)\), entonces
\[
\frac{\partial \ell}{\partial \theta_\ell}
=
\frac{\partial \ell}{\partial h_\ell}
\frac{\partial h_\ell}{\partial \theta_\ell},
\qquad
\frac{\partial \ell}{\partial h_{\ell-1}}
=
\frac{\partial \ell}{\partial h_\ell}
\frac{\partial h_\ell}{\partial h_{\ell-1}}.
\]
PyTorch construye este grafo automaticamente, pero matematicamente el proceso es
la regla de la cadena aplicada a embeddings, atencion, normalizaciones, MLPs y
cabeza lineal. Calin desarrolla este punto para redes profundas: la dificultad
no es solo escribir derivadas locales, sino organizarlas eficientemente en una
composicion grande.

\section{Lectura bibliografica aplicada al repo}

\subsection{Petersen--Zech}

La lectura de Petersen--Zech entra aqui como marco formal: datos como variables
aleatorias, hipotesis parametrizadas, riesgo poblacional, riesgo empirico,
optimizacion no convexa y Transformers como clases de funciones compuestas. En
el repo, esa teoria no se usa de forma abstracta sino asi:
\[
\calD \rightsquigarrow \text{proceso generador de texto},
\quad
\widehat R \rightsquigarrow \code{F.cross\_entropy},
\quad
\calH \rightsquigarrow
\{\text{bigram},\text{transformer},\text{llama}\}.
\]
La advertencia teorica principal es que las ventanas solapadas no cumplen la
version iid mas simple. Por eso el lenguaje de generalizacion debe usarse con
cuidado: el test loss es una estimacion empirica bajo el split del repo, no una
garantia universal.

\subsection{Wegner}

Wegner complementa el documento en tres lugares. Primero, el dataset se entiende
como una matriz de observaciones, aunque aqui las filas sean ventanas de una
secuencia dependiente. Segundo, maxima verosimilitud explica por que minimizar
NLL es equivalente a ajustar una distribucion probabilistica. Tercero, alta
dimension y compresion explican por que embeddings, SVD conceptual y
representaciones latentes pertenecen al mismo lenguaje matematico.

En notacion de datos,
\[
\mathsf X=
\begin{pmatrix}
x_0 & x_1 & \cdots & x_{B-1}\\
x_1 & x_2 & \cdots & x_B\\
\vdots & \vdots & & \vdots
\end{pmatrix},
\qquad
\mathsf Y=
\begin{pmatrix}
x_1 & x_2 & \cdots & x_B\\
x_2 & x_3 & \cdots & x_{B+1}\\
\vdots & \vdots & & \vdots
\end{pmatrix}.
\]
Esta es la matriz de diseno del experimento, con la particularidad de que sus
filas no son independientes.

\subsection{Calin}

Calin complementa las partes internas de deep learning: funciones de activacion,
costos, entropia cruzada, KL, backpropagation, informacion y capacidad. En el
repo esto aterriza en cuatro formulas:
\[
\softmax(z)_j=\frac{e^{z_j}}{\sum_k e^{z_k}},
\qquad
\ell(z,y)=-\log\softmax(z)_y,
\]
\[
\SiLU(a)=a\sigma(a),
\qquad
\mathrm{SwiGLU}(x)=W_d(\SiLU(W_gx)\odot W_ux).
\]
La lectura de capacidad no debe confundirse con tamano de parametros solamente:
dos modelos con numeros parecidos de parametros pueden tener sesgos inductivos
distintos por su forma de composicion, normalizacion y codificacion posicional.

\subsection{Chang et al.}

Chang et al. motivan la pregunta empirica: bajo restricciones de parametros, la
calidad de datos puede cambiar la eficiencia del escalamiento. En este repo, la
version operacional de esa idea es restringida:
\[
\text{calidad de datos}
\leadsto
\text{diversidad empirica, entropia local, gzip, distinct-n}.
\]
El paper no autoriza concluir que toda diversidad sea buena. La lectura correcta
para este experimento es condicional:
\[
\text{datos mas diversos son mas efectivos}
\quad\text{solo si}\quad
\text{esa diversidad reduce riesgo alcanzable bajo el presupuesto dado}.
\]
Cuando high es mas diverso pero tiene peor test loss, el resultado indica que
las metricas de diversidad capturan dificultad adicional o variacion no
aprovechada por el modelo, no que la teoria de calidad de datos sea falsa.

\section{Trazas de dimensiones en una pasada forward}

Esta seccion fija una notacion comun para leer el codigo sin ambiguedad. Sea
\[
M=\text{batch size},\qquad
T=\text{sequence length},\qquad
B=\text{block size},\qquad
m=|\calV|,\qquad
d=\text{embedding dim}.
\]
Durante entrenamiento usualmente \(T=B\), aunque durante generacion puede ser
\(T\leq B\) por el truncamiento \code{idx[:, -block\_size:]}.

\subsection{Bigrama}

Entrada:
\[
X\in\{0,\dots,m-1\}^{M\times T}.
\]
Parametro:
\[
A\in\R^{m\times m}.
\]
Salida:
\[
Z=A[X]\in\R^{M\times T\times m}.
\]
La notacion \(A[X]\) significa indexacion fila por fila: \(Z_{b,t,:}=A_{X_{b,t},:}\).
Si hay targets,
\[
Y\in\{0,\dots,m-1\}^{M\times T},
\]
la perdida aplana
\[
Z\mapsto Z^{\flat}\in\R^{MT\times m},
\qquad
Y\mapsto Y^{\flat}\in\{0,\dots,m-1\}^{MT}.
\]
Por tanto, cada posicion de cada secuencia del batch cuenta como una observacion
de clasificacion multiclase.

\subsection{Transformer pequeno}

La entrada tokenizada produce dos tablas:
\[
E[X]\in\R^{M\times T\times d},
\qquad
P[0:T]\in\R^{T\times d}.
\]
Por broadcasting,
\[
H^{(0)}=E[X]+P[0:T]\in\R^{M\times T\times d}.
\]
En una cabeza de atencion,
\[
Q,K,V\in\R^{M\times T\times d_h},
\qquad
QK^\top\in\R^{M\times T\times T},
\]
donde la transposicion es solo en las dos ultimas dimensiones. La mascara causal
tiene forma \((T,T)\) y se aplica a cada elemento del batch. La salida de una
cabeza es \(\R^{M\times T\times d_h}\). Con \(H\) cabezas,
\[
\mathrm{concat}(O^{(1)},\dots,O^{(H)})
\in\R^{M\times T\times Hd_h}
=\R^{M\times T\times d}.
\]
La proyeccion final preserva \(\R^{M\times T\times d}\). El MLP/feedforward
tambien preserva la dimension externa; solo cambia temporalmente a una dimension
oculta. Por eso cada bloque Transformer es un endomorfismo de
\(\R^{M\times T\times d}\).

\subsection{LLaMA-style}

La entrada del modelo LLaMA-style empieza igual con embeddings de token:
\[
H^{(0)}=E[X]\in\R^{M\times T\times d}.
\]
No se suma una tabla \(P_t\). La informacion posicional entra despues, al
partir cabezas:
\[
Q,K,V\in\R^{M\times H\times T\times d_h}.
\]
RoPE exige \(d_h\) par porque rota coordenadas por pares. Despues de RoPE:
\[
\widetilde Q_{b,h,t,:}=R_t Q_{b,h,t,:},
\qquad
\widetilde K_{b,h,s,:}=R_s K_{b,h,s,:}.
\]
Los scores tienen forma
\[
S\in\R^{M\times H\times T\times T}.
\]
Tras softmax y mezcla con \(V\), se obtiene
\[
O\in\R^{M\times H\times T\times d_h}.
\]
El codigo transpone y reordena:
\[
O\mapsto O'\in\R^{M\times T\times H d_h}
=\R^{M\times T\times d}.
\]
La cabeza final vuelve a
\[
Z\in\R^{M\times T\times m}.
\]

\begin{remark}[Por que estas formas importan]
Muchos errores conceptuales en Transformers vienen de confundir tres ejes:
batch, tiempo y canal. La atencion mezcla posiciones temporales, no ejemplos del
batch. Las proyecciones lineales mezclan canales, no posiciones. La mascara
actua sobre pares de posiciones, no sobre el vocabulario.
\end{remark}

\section{Errores de interpretacion que el documento debe evitar}

\begin{proposition}[Menor entropia de corpus no implica mejor corpus]
Si \(\widehat H_1(s_1)<\widehat H_1(s_2)\), no se sigue que \(s_1\) sea de
mayor calidad. Solo se sigue que la distribucion marginal de caracteres de
\(s_1\) esta mas concentrada.
\end{proposition}

\begin{proof}
La entropia marginal depende de frecuencias de caracteres. Un texto repetitivo
como \(aaaa\cdots a\) tiene entropia cercana a cero, pero no es un corpus rico.
Por tanto, la calidad no puede identificarse con baja entropia marginal.
\end{proof}

\begin{proposition}[Mayor distinct-n no implica menor perdida]
Si \(\mathrm{distinct}_n(s_2)>\mathrm{distinct}_n(s_1)\), no se sigue que un
modelo entrenado en \(s_2\) alcance menor test loss bajo igual presupuesto.
\end{proposition}

\begin{proof}
\(\mathrm{distinct}_n\) mide proporcion de n-gramas unicos. Aumentar esa
proporcion puede reducir repeticion util y aumentar la incertidumbre condicional
del siguiente token. Si el modelo no puede aprender las regularidades de largo
alcance que explican esa variedad, la perdida predictiva puede aumentar.
\end{proof}

\begin{proposition}[Perplejidad no es calidad semantica]
La perplejidad
\[
\PPL=\exp(\widehat L)
\]
es una transformacion monotona de la perdida promedio. No mide directamente
verdad, coherencia global ni valor pedagogico del texto generado.
\end{proposition}

\begin{remark}
En este repo, la perplejidad sirve para comparar modelos bajo el mismo
tokenizer, split y objetivo. Compararla entre tokenizadores distintos o corpus
con vocabularios incompatibles seria matematicamente delicado.
\end{remark}

\begin{proposition}[Un Transformer pequeno no prueba el limite de la arquitectura]
Si el \code{TinyTransformerLanguageModel} no mejora sobre el bigrama en un
experimento concreto, no se concluye que los Transformers no sirvan para el
corpus. Se concluye solamente algo sobre esta clase parametrica, esta escala y
este presupuesto de optimizacion.
\end{proposition}

\begin{proof}
La familia de Transformers contiene modelos con distintos \(d\), numero de
capas, numero de cabezas, contexto y entrenamiento. El experimento fija un punto
pequeno en esa familia. Un resultado negativo local no identifica el
comportamiento de toda la clase.
\end{proof}

\begin{remark}[Conexion con statistical learning theory]
Estas proposiciones son aplicaciones del mismo principio: una metrica observada
es una funcion parcial del fenomeno. El riesgo de test, la entropia, distinct-n
y gzip ratio son proyecciones distintas del objeto real. La inferencia rigurosa
consiste en decir exactamente que proyeccion se midio y que conclusiones no
estan autorizadas por esa medicion.
\end{remark}

\section{Limitaciones matematicas}

\begin{enumerate}
\item El modelo es character-level; no hay tokens subpalabra ni semantica
explicita.
\item Las ventanas se solapan; no hay muestra iid pura.
\item El test set es un bloque contiguo; puede haber desplazamiento temporal de
distribucion.
\item Las metricas de diversidad son locales o compresivas; no miden
syntheticity ni calidad semantica.
\item La expresividad teorica de Transformers no implica aprendibilidad con
2 capas, \(d=64\), batch size \(32\) y presupuesto limitado.
\item La comparacion medium/high es empirica y pequena; no prueba una ley
general sobre diversidad.
\end{enumerate}
